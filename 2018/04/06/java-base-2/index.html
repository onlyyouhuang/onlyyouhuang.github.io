Java基础整理

终止线程方式

  1. 使用volatile共享变量
  1. 当线程阻塞时，可以通过置中断状态取消线程，即调用interrupt()方法

排序算法稳定性

  1. 不稳定的排序算法：选择排序、希尔排序、快速排序、堆排序。
  2. 稳定的排序算法：冒泡排序、插入排序、归并排序和基数排序。

稳定排序的好处：如果多个字段需要排序，由于需要利用前面排序的结果，所以后面排序必须是稳定的，如果不稳定，则会出错。

String

  1. String类不能被继承，因为被final修饰。
  2. String类的+方法，通过构造一个新的StringBuilder对象，然后将两个String对象append()到后面。
  3. String类的hashCode被重写，通过计算每个字符的ASCII*31相加获得，这样字符串相同的放到同一个bucket内。
  4. String类的toString()被重写，返回String对象表示的字符串。

hashCode()方法中使用系数31的原因

  1. 31是一个素数，素数作用就是如果我用一个数字来乘以这个素数，那么最终的出来的结果只能被素数本身和被乘数还有1来整除！。(减少冲突)
  2. 31可以 由i*31== (i<<5)-1来表示,现在很多虚拟机里面都有做相关优化.（提高算法效率）
  3. 选择系数的时候要选择尽量大的系数。因为如果计算出来的hash地址越大，所谓的“冲突”就越少，查找起来效率也会提高。（减少冲突）
  4. 并且31只占用5bits,相乘造成数据溢出的概率较小。

char能否存储汉字

可以。因为Java中char使用utf-16进行编码，utf-16是一种变长编码，要么2个字节，要么4个字节表示。
所以当一个utf-16字符是2个字节时，则用1个char表示，如果一个utf-16字符是4个字节时，采用两个char表示。

接口能否继承多接口

接口可以继承多接口。

实现线程的3种方式

  1. 继承Thread类，重写run方法。
  2. 实现Runnable接口，重写run方法。
  3. 实现Callable接口，重写call方法。

Callable接口实际上是属于Executor框架中的功能类，Callable接口与Runnable接口的功能类似，但提供了比Runnable更加强大的功能。

  • Callable可以在任务结束的时候提供一个返回值，Runnable无法提供这个功能
  • Callable的call方法分可以抛出异常，而Runnable的run方法不能抛出异常。

Callable是类似于Runnable的接口，实现Callable接口的类和实现Runnable的类都是可被其它线程执行的任务。通过Future对象可了解任务执行情况，可取消任务的执行，还可获取任务执行的结果。

Callable和Future接口的区别

  1. Callable规定的方法是call()，而Runnable规定的方法是run().
  2. Callable的任务执行后可返回值，而Runnable的任务是不能返回值的。
  3. call()方法可抛出异常，而run()方法是不能抛出异常的。
  4. 运行Callable任务可拿到一个Future对象， Future表示异步计算的结果。它提供了检查计算是否完成的方法，以等待计算的完成，并检索计算的结果。

实现线程安全的方式

  1. 使用不可变对象。
  2. 使用ThreadLocal。
  3. 加锁。
  4. 使用无锁工具类(ConcurrentHashmap, ConcurrentLinkedQueue)。

ArrayList和LinkedList区别

  1. ArrayList数组实现。
    优点：随机访问，获取任意位置的元素 ，时间复杂度o(1)。
    添加到最后，时间复杂度o(1)。
    从中间插入，时间复杂度o(n)。
    从中间删除，时间复杂度o(n)。

  2. LinkedList是双链表实现的。
    优点：空间无限，比ArrayList耗内存。
    获取操作，时间复杂度O(n)。
    添加到尾部，时间度杂度o(1)。
    删除操作，时间复杂度o(1)，因为是双链表。

Java 移位运算

  1. <<：带符号左移，无论正数还是负数， 移动后右侧都补0。
  2. >>：带符号右移， 正数，移动后左侧补0， 负数移动后左侧补1。
  3. >>>：无符号右移，移动后，左侧不足补0。

抽象类

  1. 抽象类中可以有构造方法。
  2. 抽象类中可以有普通属性，方法，静态属性和方法
  3. 抽象类中可以有抽象方法
  4. 如果一个类中有抽象方法，那么一定是抽线类。一个抽象类中可以没有抽象方法。
  5. 抽象类中的方法必须由子类实现，如果子类没有实现，则子类也是一个抽象类。

接口

  1. 接口中只有方法的声明，没有方法体。
  2. 接口中可以有常量，因为会被默认加上public static final。
  3. 接口中的方法必须是public的
  4. 接口中没有构造方法，也不能实例化。
  5. 接口可以实现多继承。
  6. 接口中定义的类必须全部实现，如果实现类没有全部实现，则实现类定义为抽象类。

try catch

这里finally中的返回值会覆盖try中的返回值，所以返回1。

java 异常类

运行时异常包括(常见的)：

  1. NullPointerException(空指针异常)
  2. ClassNotFoundException(类未找到异常)
  3. IndexOutOfBoundsException(数组越界异常)

IO异常(常见的)：

  1. EOFException：已经读到末尾了
  2. FileNotFoundException：文件未找到。

VirtualMachineError，包括：

  1. StackOverFlowError
  2. OutOfMemoryError

final修饰的基本数据类型和类型转化

  1. final修饰的基本数据类型是常量
  2. 在基本运算操作中，byte，short的运算结果，会被向上转型为int，float也会向上转型为double，所以将运算结果赋值时，需要做强制类型转化，否则编译不通过。

jvm参数

  1. -Xms：初始化堆的大小，堆内存的最小值
  2. -Xmx：堆最大内存大小
  3. -Xmn：年轻代的大小

super()

如果在子类的方法中需要调用父类的构造方法，则可以使用super()，而且super()一定要写在子类构造方法的第一行。

类权限修饰符

public > protected > package > private。
所以从上面可以看出protected包含包访问权限。

接口继承

一个接口可以继承多个接口。
但是一个类只能继承一个接口。

反射的作用

  1. 在运行期间判断一个对象所属的类。
  2. 在运行期间构造一个类的对象。
  3. 在运行期间判断一个类具有的成员变量和方法。
  4. 在运行期间调用一个对象的方法。

static变量

static变量不能用this引用。

静态内部类

静态内部类不可以访问外部类的非静态方法和成员变量。
非静态内部类可以直接访问外部类的的数据，包括私有数据。

关于System.out.println()

  1. System.out.println(100 + 5 + “is”) 打印105is。
  2. System.out.println(“is” + 100 + 5) 打印is1005。

重写(覆盖)方法的约束

  1. 方法名，参数列表相同。
  2. 子类的返回值类型小于等于父类的返回值类型。
  3. 子类声明抛出的异常小于等于父类声明抛出的异常。
  4. 子类的权限大于等于父类的权限修饰符。

基本数据类型的转换和计算

低——————————————————->高
byte,short,char -> int -> long -> double -> float -> double

关于继承中无参构造方法

用new创建子类的对象时，若子类没有带参的构造方法，则先执行父类的无参构造方法，然后再执行自己的构造方法。

父类的私语变量和方法

子类会继承父类的私有变量和方法，只是不能访问。

Integer的valueOf和直接int赋值

Integer直接int赋值和valueOf(int)获取的Integer对象时：

  1. 如果int数值，在-128~127之间，则直接返回同一个对象(缓存对象)。
  2. 如果int数值的数值超过这个范围，则新建一个对象。

继承和实现的顺序

一个类如果同时继承一个类，和实现多个接口，则继承和实现的顺序：
继承在实现之前。

外部类的访问修饰符

外部类的修饰符只能是public和默认包权限。

abstract关键字的使用

abstract关键字不可以与final、static、synchronized一起使用。
但是可以与protected一起使用。

枚举

  1. 从jdk1.6开始 switch支持枚举
  2. 枚举创建是线程安全的

枚举类使用enum定义后在编译后默认继承了java.lang.Enum类，而不是普通的继承Object类。enum声明类继承了Serializable和Comparable两个接口。且采用enum声明后，该类会被编译器加上final声明（同String），故该类是无法继承的。枚举类的内部定义的枚举值就是该类的实例（且必须在第一行定义，当类初始化时，这些枚举值会被实例化）。

String s=”abc”创建多少个对象

  1. String s=new String("abc")，这条语句会创建2个对象，分别是"abc"会在常量池中创建一个对象，new又回在堆里面创建一个对象。
  2. String s="abc"会在常量池中创建一个对象。
  3. String s="ab"+"cd"会创建三个对象，即首先在常量池创建ab和cd两个对象，最后在常量池中创建abcd这个对象。

所以，当执行String s="abc"时，首先会在常量池中寻找”abc”是否在常量池中，如果不再则在常量池中创建一个"abc"，否则则存在直接获取这个字符串对象。

接口访问权限

接口访问权限只能为public或者default。
接口方法只能为为public。

abstract访问权限

  1. abstract不能与final一起修饰一个类。
  2. abstract不能与private、static、final或者native并列修饰一个方法。
