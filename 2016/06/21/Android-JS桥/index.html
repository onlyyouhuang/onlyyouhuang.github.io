Android JS桥

  • Android
  • Html5

Android项目中部分对性能要求不是高的页面会使用H5来实现，这样的页面也可以在IOS平台上使用，这里就涉及了Java和js的交互。

JavascriptInterface

Android Webview自带的JavascriptInterface，可以通过addJavascriptInterface将一个对象加入到webview中作为js方法供js调用，从而实现js与Java的通信。

在4.0之前要使得WebView加载js只需如下代码：

mWebView.addJavascriptInterface(new JsToJava(), "myjsfunction");

这句代码在android4.2版本以上会报错

提示None of the methods in the added interface have been annotated with @Android.webkit.JavascriptInterface; they will not be visible in API 17

解决办法：在调用方法上面加上@JavascriptInterface注解，如果代码无此申明，那么也就无法使得js生效，也就是说这样就可以避免恶意网页利用js对安卓客户端的窃取和攻击。

class JsToJava {
    @JavascriptInterface
    public void XXX() {...}
}

在Html页面的script中，使用如下代码调用：

window.myjsfunction.XXX();

JsBridge

JsBridge(IOS端为WebViewJavascriptBridge)是github上的一个框架，用作者的话来说就是实现java（IOS端为Obj-C）和js的互相调用的桥梁。替代了WebView的自带的JavascriptInterface的接口，使得我们的开发更加灵活和安全。

Android Studio中引入到工程：

repositories {
    // ...
    maven { url "https://jitpack.io" }
}

dependencies {
    compile 'com.github.lzyzsd:jsbridge:1.0.4'
}
