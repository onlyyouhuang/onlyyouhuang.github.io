淘宝SDK for Android

  • Android
  • SDK
  • remoteService

公司的App初期需要调用淘宝SDK来完成物品详情查看和支付购买，淘宝开放平台提供了相关的功能，现做一整理。

淘宝开放平台

开发者在开发应用前必须先到淘宝开放平台（TOP,Taobao Open Platform）申请接入，获取appkey和appsecret，同时需要通过无线事业部分配相应ttid之后才可以正常调用开放平台的接口以及组件。

  • ttid：淘宝无线事业部分配给开发者的特定参数，用于客户端应用识别，及相关数据统计与分析
  • appkey：淘宝开放平台颁发的唯一标识一个应用的ID号，该appkey可在淘宝开放平台－合作伙伴后台创建应用后获取
  • appSecret：应用密钥，不可对外公开，商家需注意保密，调用API的时候需要使用，可以通过该密钥获得的商家所有数据

摩宝top sdk for Android

摩宝top sdk for Android完全遵循淘宝top规则，与top sdk for Java调用方式一致。
官方提供的demo是结合使用淘宝Android数据服务中心后台服务组件。

淘宝Android数据服务中心

淘宝Android数据服务中心(TaoBao Android Data and Service Center)是淘宝基于Android平台开发的一个独立的后台服务组件，主要用来向第三方应用程序提供淘宝购物时所需要的基本的数据服务， 数据中心为第三方开发者提供统一的服务调用方式以及消息回传机制。
组件服务包含：

  • 统计服务
  • 登录服务
  • 登出服务
  • 商品详情服务(内嵌收货地址服务、用户订单服务、交易支付服务)

SDK开发包中包含:

  • tbDataService.apk（数据服务中心安装包）
  • tbDataService.jar(数据服务中心对外调用接口jar包，集成时使用)

捆绑安装

淘宝为第三方应用客户端提供了集成数据服务中心的方式为捆绑安装，即将数据服务中心安装包apk与第三方应用客户端整合在一起。捆绑安装的具体步骤如下：

  1. 将tbDataService.apk作为资源复制到第三方应用工程中的assets目录。
  2. 第三方应用在需要数据中心服务时（或者其它恰当时机），先检测数据服务中心是否已经安装。如果尚未安装，则从assets目录中提取tbDataService.apk到手机存储。
  3. 安装手机存储中的数据服务中心apk。

AIDL

使用tbDataService.apk提供的服务，这里就需要在一个进程中(例如在一个Activity中)访问另一个进程中(例如一个Service)某个对象的方法。
AIDL(android接口描述语言)是一个IDL语言，它可以生成一段代码，可以使在一个android设备上运行的两个进程使用内部通信进程进行交互。Service以aidl文件的方式提供服务接口，AIDL工具将生成一个相应的Java接口，并且在生成的服务接口中包含一个功能调用的stub服务桩类。Service的实现类需要去继承这个stub服务桩类。Service的onBind方法会返回实现类的对象，之后就可以使用它了。

remoteService

使用AIDL实现就可以在客户端中调用服务端的方法，流程如下：

  1. Client：Intent intent = new Intent(“类名”)构建intent，然后bindService(intent, serviceConnection, Context.BIND_AUTO_CREATE)绑定服务；
  2. Client：通过ServiceConnection()重载onServiceConnected()建立连接，获取Service.Stub实例，onServiceDisconnected()释放连接；
  3. Service：重载onBind(Intent intent)方法，返回Service.Stub实例，但Service.Stub类是由aidl文件生成的接口类中的一个内部类Stub类，Service来继承该Stub类，然后执行onCreate()函数（注：bindService不执行onStart()函数）；
  4. Client：通过操作Service实例，调用服务端接口。

callback

如果要从服务端去调用客户端的方法，那么就需要注册callback，流程如下：

  1. 建立两个AIDL文件：一是用于存放供给Client端调用的方法（里面包括注册回调方法的接口），二是用于存放要回调Client端的方法；
  2. Client：实现回调方法，并在ServiceConnection()的onServiceConnected()获取Service.Stub实例后，调用注册回调方法的接口；
  3. Service：新建RemoteCallbackList对象(一个存储回调对象的列表)，这样服务器端才知道哪些函数可以调用;
  4. Service：通过操作RemoteCallbackList对象进行回调Client端的方法。

AppLinkSDK

当需要从我们的app跳转到手淘完成相关操作，可使用该SDK。开发前需要导入AppLinkSDK.jar包，功能包括跳转授权、店铺、详情、h5页面，并提供渠道优惠、淘客的结算。
当手淘里的操作结束，需要从手淘跳转回我们的app时，手淘通过url启动相应的activity，因此在调用接口时要传入backurl的值。
假设MainActivity的url为tbopen:m.sdk.com，在AndroidManifest.xml中对应的activity节点下对action和data等标签进行如下配置即可

<activity
    android:name=".activity.MainActivity"
    android:label="@string/app_name">
    <intent-filter>
        <action android:name="android.intent.action.VIEW" />
        <category android:name="android.intent.category.DEFAULT" />
            <data
                android:pathPrefix="/index.html"
                android:host="m.sdk.com"
                android:scheme="tbopen" />
    </intent-filter>
</activity>

scheme, host, port, path, pathPrefix, pathPattern 是用来匹配 Intent 中的 Data Uri 的。具体规则如下:

scheme://host:port/path or pathPrefix or pathPattern

最后的 “path or pathPrefix or pathPattern” 是指后面的 path 验证可以使用 data 属性中的 android:path、android:pathPrefix 或 pathPattern，你可以添加任意个 data 标签，由于是 “or” ，因此，只要其中任意一个 data 匹配，系统就会选择你的 Activity 启动，当然，如果别的 Activity 也有相同的 data 标签，系统就会给用户弹出一个 Chooser Dialog。
mimeType也是是用来匹配 Intent 的。比如，当你使用 Intent.setType(“text/plain”) ，那么系统将会匹配到所有注册 android:mimeType=”text/plain” 的 Activity。
