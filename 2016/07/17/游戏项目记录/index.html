游戏项目记录

  • Android
  • Java

记录一个参与的游戏项目中实现交互模块的解决方案。

最初的需求是要实现一个Android手机遥控器，可以向Android设备上的应用发送指令。先是借鉴聊天室，客户端和服务器端建立Socket连接发送消息。但是这样使用上需要先安装一个Android应用，不太方便。因此考虑将客户端改用H5网页的形式实现，改用WebSocket建立连接，顺便解决了iOS用户使用的问题。

然后了解到了WebSocket的存在。

WebSocket协议是基于TCP的一种新的网络协议，在2008年诞生，2011年成为国际标准。所有浏览器都已经支持了。
它的最大特点就是，服务器可以主动向客户端推送信息，客户端也可以主动向服务器发送信息，是真正的双向平等对话，属于服务器推送技术的一种。
其他特点包括：
（1）建立在 TCP 协议之上，服务器端的实现比较容易。
（2）与 HTTP 协议有着良好的兼容性。默认端口也是80和443，并且握手阶段采用 HTTP 协议，因此握手时不容易屏蔽，能通过各种 HTTP 代理服务器。
（3）数据格式比较轻量，性能开销小，通信高效
（4）可以发送文本，也可以发送二进制数据。
（5）没有同源限制，客户端可以与任意服务器通信。
（6）协议标识符是ws（如果加密，则为wss），服务器网址就是 URL。

目前有很多Java实现，项目里使用了Java-WebSocket

A barebones WebSocket client and server implementation written in 100% Java. http://java-websocket.org/       
可以用在JAVA 1.5和Android 1.6之上的WebSocket解决方案，客户端和服务器端都可以实现。

H5网页还是内嵌在客户端的WebView，如果再另外写个后台程序提供H5页面就太麻烦了。最后找到了NanoHttpd 

Tiny, easily embeddable HTTP server in Java. http://nanohttpd.org  
仅仅用一个Java类，就实现了一个轻量级的 Web Server，可以非常方便地集成到Android应用中去
让你的App支持 HTTP GET, POST, PUT, HEAD 和 DELETE 请求。

它使用的是Socket BIO(Blocking I/O，同步阻塞I/O)，一个客户端连接分发到一个线程的经典模型，而且具有良好的扩展性。之后需要再记录一篇关于I/O模型的博客。
它有两个弊端：

1，读和写操作是同步阻塞的，任何一端出现网络性能问题，都会影响另一方。
2，一个链路建立一个线程，无法满足高并发，高性能需求。

因为每次加入游戏的玩家数量上限是6个，能够满足我们的需求。借助NanoHttpd直接实现一个简易的Web Server，嵌入到游戏端。这样游戏端放出一个二维码，手机用浏览器或内嵌浏览器的应用扫一扫就可以直接得到连接地址，与游戏端建立连接，并从Web Server获取到H5页面。
