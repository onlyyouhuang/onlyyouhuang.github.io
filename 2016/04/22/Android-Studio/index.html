Android Studio

  • Android

搬家到Android Studio啦，说说使用感受。

因为15年选择了辞职考研二战，因此在技术学习方面也成为了空白的一年。恰巧Android Studio在15年算是在Android开发领域站稳了脚步，大部分开发者纷纷弃eclipse投入到AS的大队伍中(至少我身边做安卓开发的朋友是这样)。今年折腾完考研的事情后，趁开学前的这段时间啃老本找了Android开发的实习，发现自己的开发经验已经完全是老古董了，不得不感慨技术发展的速度。在恶补一番之后，谈谈这段时间AS的使用感想。

现在开始使用AS还算有点好处，就是AS已经比较稳定，没有经历那段不成熟时的各种问题，别人走过的坑在网上有过总结，很多问题也可以从stackoverflow上找到解决方案。

AS还是好用的。首先长得挺好看（-皿-），这就是个看脸的时代啊。我选了自带的黑色主题，也可以到什么网（忘了）下载一个定制的主题。码代码方面，代码提示要更全面更智能些，默认的快捷键不熟悉的话可以在设置里改成eclipse的习惯。里面自带的插件，像git什么的也是挺全的。自动保存，这点赞啊，不用边写边ctrl+s，即使文件关闭了，只要工程没有被关闭，就还可以不断回退到刚打开时的代码。挺多优点在使用过程中会不断发现，也就不一一罗列了。

再讲讲我在使用过程中不太方便的地方。

  • 导入不方便

    • 导入三方类库
      可以通过在module的build.gradle中dependencies添加’compile … ‘快速引入。或是以module的形式将类库导入工程，例如eclipse的一些lib工程，可以参考如下步骤。
      （1）首先在lib工程目录下添加build.gradle：

      apply plugin: 'android-library'
android {
    compileSdkVersion 23
    buildToolsVersion "23.0.3"
    sourceSets {
        main {
            manifest.srcFile 'AndroidManifest.xml'
            java.srcDirs = ['src']
            resources.srcDirs = ['src']
            aidl.srcDirs = ['aidl']
            renderscript.srcDirs = ['src']
            res.srcDirs = ['res']
            assets.srcDirs = ['assets']
        }
    }
}

      （2）在app/build.gradle下添加编译依赖：

      dependencies {
    compile fileTree(include: ['*.jar'], dir: 'libs')
    compile project(':XXX')
}

      （3）项目根目录下的settings.gradle文件中include后面添加第三方库目录：

      include ':app', ':XXX'

      （4）最后Sync一下就OK了。
      这里也顺便记录下，eclipse导入library工程的步骤：
      （1）file -> new -> other -> android类下的androd project from existing code,工程包里面有个library文件夹，选择那个文件夹导入。
      （2）或是直接import -> Android -> Existing Android Code Into Workspace选择工程包，把里面的sample、library都一起导进去。

    • 导入工程
      这里指AS的工程。为了更快地完成导入，在导入工程之前把以下几个文件里的内容改成目前正在使用的版本，这样不用再另外下载了：
      （1）根目录下的build.gradle，dependencies里classpath ‘com.android.tools.build:gradle:X.X.X’，这里gradle的版本号；
      （2）与第1处的gradle版本对应的在gradle\wrapper\gradle-wrapper.properties里的distributionUrl；
      （3）每个module（一般就只有一个叫app的module）里的build.gradle，

      android {
    compileSdkVersion 23
    buildToolsVersion "23.0.3"
    ...
}

      改好之后，就可以import project了。有些工程可能只支持老的版本，比如开源中国的开源客户端，折腾了一会儿之后放弃了，直接import工程，然后根据报错的提示下载本地缺少的编译包。

    • 编译慢
      编译比较大的工程时，Gradle Build Running挺久的，而且占用的内存就会直接飙到90+，结束后就马上恢复正常，当然也有我的笔记本年事已高的原因。
      Android Studio 2.0版引入热代码替换技术，加入了即刻预览功能Instant Run。一个代码的变动能立刻（1-2秒）在连接的物理设备或者模拟器上生效，并且不需要通过一个完整的或者增量的完整编译和APK开发工具就能执行这个操作。这个改进就会使代码小改动的时候变得很方便。

可能碰到的报错：

  • 报错一： 

    Error:Execution failed for task ':myapp:dexDebug'. com.android.ide.common.process.ProcessException: org.gradle.process.internal.ExecException: Process 'command '%JDK_HOME%/bin/java'' finished with non-zero exit value 2

    有可能引入了重复包。例如我一开始引入了com.android.support:support-v4，之后又引入一个里面包含support-v4的三方类库，编译的时候就会报这个错
