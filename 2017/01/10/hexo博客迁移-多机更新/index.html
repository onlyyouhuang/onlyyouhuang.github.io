记录hexo博客迁移和多级更新过程

为了等MBP2016发布，旧笔记本修修补补又可以再战几年。趁空挡时间找的实习工作，就是为了能在第一时间入手MBP（哭）。

那么，换了新的笔记本，如何继续更新hexo博客？
首先同样需要在新笔记本安装好git（Mac自带）和Node.js，做好git和GitHub的配置工作。然后将旧笔记本blog的文件拷贝到新电脑中。

另起炉灶

以下文件是需要的：

_config.yml
theme/
source/
scaffolds/
package.json
.gitignore，

以下文件需要删除

.git/
node_modules/
public/
.deploy_git/
db.json

在终端cd到该站点文件夹，执行下列命令：

npm install

这里不需要使用hexo init进行初始化，因为这部分文件已经拷贝过来了，如果执行了配置文件_config.yml里面内容会被清空使用默认值，所以注意这里不要执行hexo init。

之后就可以使用hexo g，然后使用hexo d进行部署，如果都没有出错，就转移成功了。

多机更新

上面的步骤只适用于单机更新博客的情况，如果要多机更新，这里引用一下高票答案的流程：

1. 创建仓库，http://CrazyMilk.github.io；
2. 创建两个分支：master 与 hexo；
3. 设置hexo为默认分支（因为我们只需要手动管理这个分支上的Hexo网站文件）；
4. 使用git clone git@github.com:CrazyMilk/CrazyMilk.github.io.git拷贝仓库；
5. 在本地http://CrazyMilk.github.io文件夹下通过Git bash依次执行npm install hexo、hexo init、npm install 和 npm install hexo-deployer-git（此时当前分支应显示为hexo）;
6. 修改_config.yml中的deploy参数，分支应为master；
7. 依次执行git add .、git commit -m "..."、git push origin hexo提交网站相关的文件；
8. 执行hexo g -d生成网站并部署到GitHub上。

前四步的目的是，在github上建立一个新的repo，并且把目录同步到本地（这时同步到本地的文件夹可能只有一个readme.md，或者是个空文件夹，但是有一个隐藏的.git文件夹。之所以要clone一个空repo，这里是要通过git建立存放本地环境的文件夹。
执行第五步的时候，Hexo会生成一个新的.git，并且覆盖了上文提到的.git，这会导致没法push到hexo分支。正确的做法是：在hexo init前复制.git，在完成hexo init后再黏贴回来覆盖新生成的.git，剩下的就按照高票答案就能完成push hexo了。
